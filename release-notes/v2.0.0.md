Release Notes — Version 2.0.0
Release Date: April 15, 2026
Type: Major release
> **Important:** Version 2.0.0 includes breaking changes from v1. See the
> [Migration Guide](#migration-guide) below before upgrading.
---
What's New
Redesigned API Structure
Version 2.0.0 introduces a redesigned API with cleaner endpoint naming, a
consistent response format across all endpoints, and improved error messages.
New Base URL
The API base URL has changed from `https://api.flowsync.io/v1` to
`https://api.flowsync.io/v2`. Update all your API calls to use the new URL.
Workspace-Scoped Resources
All resources are now scoped to a workspace. This enables multi-workspace
support for enterprise accounts. Your API key determines which workspace
your requests operate in.
New Error Response Format
Error responses now follow a consistent JSON structure across all endpoints:
```json
{
  "error": {
    "code": "error_code",
    "message": "Human-readable description",
    "field": "field_name"
  }
}
```
---
Breaking Changes
Endpoint Renames
v1 Endpoint	v2 Endpoint
`GET /v1/flows`	`GET /v2/workflows`
`POST /v1/flows`	`POST /v2/workflows`
`GET /v1/flows/{id}`	`GET /v2/workflows/{workflow_id}`
`PUT /v1/flows/{id}`	`PUT /v2/workflows/{workflow_id}`
`DELETE /v1/flows/{id}`	`DELETE /v2/workflows/{workflow_id}`
Field Renames
The `id` field on all resources has been renamed to the resource-specific ID
field — for example, `workflow_id`, `trigger_id`. Update any code that
references the `id` field.
---
Migration Guide
To migrate from v1 to v2:
Update the base URL in all API calls from `https://api.flowsync.io/v1`
to `https://api.flowsync.io/v2`.
Update endpoint paths using the rename table above.
Update any code that references the `id` field to use the new resource-specific
ID field (`workflow_id`, `trigger_id`).
Update error handling code to use the new error response format.
The v1 API remains available until October 1, 2026. Migrate to v2 before
this date to avoid service interruptions.
